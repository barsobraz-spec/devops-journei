## Команды которые выполнил

```bash
# Создание логов
echo "app_name=coffee-bot" > logs/app.log
echo "error:connection failed" >> logs/app.log
echo "error:timeout" >> logs/app.log

# Поиск
grep "error" logs/app.log
grep -i "ERROR" logs/app.log
grep -v "error" logs/app.log
grep -n "error" logs/app.log
grep -c "error" logs/app.log

# Pipe и подсчёт
grep -v "200" logs/nginx.log | wc -l
```

## Что понял

### grep = поиск текста
- `grep "error" file` — найти все строки с "error"
- Работает как фильтр: пропускает только нужное

### Флаги grep
- `-i` — игнорировать заглавные буквы (error, ERROR, Error)
- `-v` — показать всё КРОМЕ слова (исключить)
- `-n` — показать номер строки (зачем открывать файл на 10000 строк)
- `-c` — считать количество строк (быстро понять масштаб)

### pipe (|) = соединение команд
- `cat file | grep "error"` — результат cat идёт в grep
- `grep "error" file | wc -l` — grep → wc считает строки
- Полезен когда нужно связать несколько фильтров

### Практический пример
В nginx логе 6 запросов:
- 3 успешных (200)
- 3 ошибки (404, 500, 403)

```bash
grep "200" logs/nginx.log      # 3 строки
grep -v "200" logs/nginx.log | wc -l  # результат: 3
```

## Где пригодится в production

- Искать ошибки в /var/log/nginx.log
- Считать сколько 500 ошибок за сегодня
- Исключать debug логи: `grep "error" app.log | grep -v "debug"`
- Найти проблему на конкретной строке через -n
